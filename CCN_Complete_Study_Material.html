Data Communication — Definition & Components

DEFPYQ ▾
DEFINITION Data communication is the exchange of data between two devices via some form of transmission medium (e.g., wire/cable). The communicating devices must be part of a communication system made of hardware and software.

Five Characteristics of Effective Data Communication

  • Delivery — Data must reach the correct destination only
  • Accuracy — Data must be delivered without alteration
  • Timeliness — Data must be delivered on time (real-time for audio/video)
  • Jitter — Variation in packet arrival time (uneven delay in audio/video)

Five Components of a Data Communication System

┌─────────┐ Protocol ┌──────────────┐ Protocol ┌──────────┐ │ SENDER │ ─────────────▶ │ MEDIUM │ ─────────────▶ │ RECEIVER │ └─────────┘ │(Twisted Pair │ └──────────┘ │ Coax, Fiber │ │ Radio waves)│ └──────────────┘ ◀─────────────── MESSAGE (text/audio/video/image) ──────────────▶
  • Message — The information to be communicated (text, numbers, images, audio, video)
  • Sender — Device that sends the data (computer, phone, camera)
  • Receiver — Device that receives the message (computer, TV, phone)
  • Transmission Medium — Physical path (twisted-pair, coaxial, fiber-optic, radio waves)
  • Protocol — Set of rules that govern data communications
PROTOCOL (Most Asked — 1 Mark) A protocol is a set of rules that govern data communications. It represents an agreement between the communicating devices. Without a protocol, two devices may be connected but not communicating.

Data Representation Forms

  • Text — Represented as bit patterns (ASCII uses 7-bit; Unicode uses 32-bit)
  • Numbers — Directly converted to binary (not ASCII-coded)
  • Images — Matrix of pixels; each pixel assigned a bit pattern (RGB for color)
  • Audio — Continuous signal; converted from sound via microphone
  • Video — Continuous or combination of discrete images (frames)

SIMPLEX Communication is unidirectional. Only one device on a link can transmit; the other can only receive. Uses entire channel capacity for one direction.
HALF-DUPLEX Each station can both transmit and receive, but NOT at the same time. When one device sends, other can only receive. Entire capacity taken over by transmitting device.

OSI Model — 7 Layers (Full Diagram & Explanation)

IMP 7MPYQ ▾
OSI = Open Systems Interconnection Developed by ISO (International Organization for Standardization). A layered framework for the design of network systems that allows communication between all types of computer systems. It is NOT a protocol — it is a MODEL. Has 7 separate but related layers.

Physical · Data Link · Network · Transport · Session · Presentation · Application

Layer-by-Layer Explanation

LAYER 1 — PHYSICAL Deals with physical/electrical/mechanical specifications. Responsible for bit transmission (0s and 1s) over medium. Defines data rate, synchronization, topology, transmission mode. Devices: Hub, Repeater, Ethernet cable. PDU: Bit.
LAYER 2 — DATA LINK Node-to-node frame delivery. Handles framing, physical (MAC) addressing, flow control, error control, access control. Divided into LLC and MAC sublayers. Devices: Switch, Bridge. PDU: Frame.
LAYER 3 — NETWORK Source-to-destination packet delivery. Provides logical (IP) addressing, routing, packetization. Devices: Router. PDU: Packet/Datagram.
LAYER 4 — TRANSPORT Process-to-process message delivery. Service point (port) addressing, segmentation, reassembly, flow & error control, connection control. Protocols: TCP, UDP. PDU: Segment.
LAYER 5 — SESSION Establishes, controls, maintains user sessions. Dialogue control and synchronization. Ensures sessions are properly opened/closed.
LAYER 6 — PRESENTATION Translation (converts data formats), Encryption/Decryption, Compression. Acts as a translator between network and application.
LAYER 7 — APPLICATION Provides user interface to access network services. Protocols: HTTP, FTP, SMTP, DNS, Telnet, SNMP. Process-to-process communication.

TCP/IP Protocol Suite — Diagram & All Protocols

IMP 7MPYQ ▾
TCP/IP A protocol suite (set of protocols organized in layers) used in the Internet. Hierarchical protocol — each upper-level protocol supported by one or more lower-level protocols. Today considered a 5-layer model.

TCP/IP Layer Descriptions

  • Physical Layer — Carries individual bits across the link; electrical/optical signals
  • Data Link Layer — Takes datagram and moves it across a link; encapsulates in frame; supports all standard protocols
  • Network Layer — Host-to-host communication; IP defines packet format and routing
  • Transport Layer — End-to-end logical connection; TCP (reliable, connection-oriented) or UDP (unreliable, connectionless)
  • Application Layer — Process-to-process communication; HTTP, FTP, SMTP, DNS, Telnet

Network Topologies (PYQ 7 Marks)

MESH TOPOLOGY Every device has a dedicated point-to-point link to every other device. For n nodes: n(n-1)/2 duplex links needed. Advantages: Dedicated links (no traffic issues), robust (if one link fails, system continues), privacy, easy fault isolation. Disadvantages: Expensive (lots of cabling), difficult installation.
MESH (5 nodes) STAR BUS RING A—B—C [DEV1][DEV2] Node1 Node2 Node3 [S1] |\ |\ | \ / | | | | | \| \| [HUB] ──●──────●──────●── [S2] D—E—F [DEV3][DEV4] Cable end Cable end | [S3]
STAR TOPOLOGY Each device has dedicated link only to a central controller (HUB). No direct device-to-device links. Hub acts as exchange. Advantages: Less expensive, easy install/reconfigure, fault isolation easy, one link failure only affects that link. Disadvantages: If hub fails, whole system fails; more cabling than ring.
BUS TOPOLOGY Multipoint topology. One long cable (backbone) connects all devices via drop lines and taps. Advantages: Easy installation, less cabling than mesh/star. Disadvantages: Difficult reconnection/fault isolation, hard to add new devices, signal reflection at taps.
RING TOPOLOGY Each device has point-to-point connection with only TWO adjacent devices. Signal passes in one direction. Each device has a repeater. Advantages: Easy install/reconfigure (only 2 connections to change), fault isolation easy. Disadvantages: Unidirectional traffic, break in ring disables entire network.

Physical Layer — Concept & Functions

DEFPYQ ▾
PHYSICAL LAYER The bottom-most layer in the OSI Model. A physical and electrical representation of the system. Responsible for communication of unstructured raw data streams over a physical medium. PDU: Bit.

Functions of Physical Layer

  • Data Rate — Maintains how many bits/second a sender can send
  • Synchronization — Synchronizes bits between sender and receiver
  • Transmission Medium Decision — Direction of data transfer
  • Physical Topology — Decides network layout (Mesh, Star, Bus, Ring)
  • Physical Medium & Interface — Provides physical medium and interface decisions
  • Configuration — Point-to-Point and Multi-Point configuration
  • Modulation — Converting data into radio waves (electrical/optical signal)
  • Switching — Data packets forwarded from sender port to destination port
  • Devices used: Hubs, Ethernet cables, Repeaters

Transmission Impairment & Its Causes (PYQ)

┌─────────────────────────┐ │ TRANSMISSION IMPAIRMENT│ └────────────┬────────────┘ │ ┌──────────┼──────────┐ ▼ ▼ ▼ ATTENUATION DISTORTION NOISE
ATTENUATION Loss of energy — the strength of the signal decreases with increasing distance which causes loss of energy. (Signal weakens as it travels.)
DISTORTION Change in the shape/form of the signal. Occurs in composite signals made of different frequencies. Each frequency component has its own propagation speed → reaches at different times → delay distortion.
NOISE Unwanted signal added to data during transmission. Types: Thermal noise (random motion of electrons), Induced noise (motors/appliances), Crosstalk (effect of one wire on another), Impulse noise (power lines/lightning).

Transmission Media — Guided & Unguided

IMP 7MPYQ ▾
TRANSMISSION MEDIA A communication channel that carries information from sender to receiver. Data transmitted through electromagnetic signals. It is a physical path between transmitter and receiver. Available at Physical Layer (Layer 1) of OSI.
┌──────────────────────────┐ │ TRANSMISSION MEDIA │ └────────────┬─────────────┘ │ ┌──────────┴──────────┐ ▼ ▼ GUIDED UNGUIDED (Wired) (Wireless) │ │ ┌──┴──────────┐ ┌──┴──────────┐ │Coaxial │ │Radio waves │ │Fiber Optic │ │Microwaves │ │Twisted Pair │ │Infrared │ └─────────────┘ └─────────────┘

1. Twisted-Pair Cable

  • Uses metallic (copper) conductors
  • Two conductors twisted together (each with plastic insulation)
  • One wire carries signal; other used as ground reference
  • Most common connector: RJ-45
  • Used in: telephone lines, Local Area Networks
  • Twisting reduces interference with adjacent pairs
  • Types: UTP (Unshielded) and STP (Shielded)

2. Coaxial Cable

  • Uses metallic (copper) conductors; accepts electrical signals
  • Central core conductor of solid wire (copper) enclosed in insulating sheath
  • Encased in outer conductor of metal foil (shield against noise)
  • Protected by plastic cover
  • Most common connector: BNC (Bayonet Neill-Concelman)
  • Used in: telephone lines, cable TV networks, Ethernet LANs

3. Fiber-Optic Cable (PYQ 7 Marks)

  • Made of glass or plastic; transmits signals in the form of light
  • Uses principle of total internal reflection
  • Glass/plastic core surrounded by cladding of less dense glass/plastic

Advantages:

  • Higher bandwidth
  • Less signal attenuation
  • Immunity to electromagnetic interference
  • Resistance to corrosive materials
  • Lightweight
  • Greater immunity to tapping (more secure)

Disadvantages:

  • Expensive installation and maintenance
  • Unidirectional light propagation (need two cables for full-duplex)
  • High cost

Data Link Layer — Services, Framing, Flow & Error Control

IMP 7MPYQ ▾

Data Link Layer Services (PYQ)

  • Framing — Breaking bit stream into manageable frames
  • Physical Addressing — MAC address added to frame header
  • Flow Control — Prevents fast sender from overwhelming slow receiver
  • Error Control — Detection and retransmission of damaged/lost frames
  • Access Control — Determines which device has control at any time

Framing (PYQ 3 Marks & 7 Marks)

FRAMING Data link layer receives raw bit stream from physical layer. To ensure reliable transfer to network layer, it breaks the bit stream into frames, computes checksum for each frame, and transmits frame with checksum.

Parts of a Frame:

  • Frame Header — Contains source and destination addresses
  • Payload field — Contains the message to be delivered
  • Trailer — Contains error detection/correction bits (CRC)
  • Flag — Marks beginning and end of frame

Types of Framing:

  • Fixed-sized Framing — Frame size is fixed; length acts as delimiter; no extra bits needed for start/end identification
  • Variable-sized Framing — Size differs; needs mechanisms to mark end of one frame and beginning of next
    • Byte-Stuffing (Character-oriented) — A special byte stuffed in message to differentiate from delimiter
    • Bit-Stuffing (Bit-oriented) — A pattern of bits of arbitrary length stuffed in message

Flow Control (PYQ 1 Mark & 3 Marks)

FLOW CONTROL The process of managing the rate of data transmission between two nodes to prevent a fast sender from overwhelming a slow receiver. If sender sends too fast, receiver may be overloaded and data may be lost.

Two Mechanisms:

  • Stop and Wait — Sender transmits one frame, then stops and waits for acknowledgement before sending next frame. Simple but wastes resources (sender idle while waiting).
  • Sliding Window — Both sender and receiver agree on a number of frames. Sender can send multiple frames before waiting for ACK. More efficient, uses underlying resources better.

Error Control (PYQ)

ERROR CONTROL Mechanism to detect and correct errors in data frames. Used when data-frame may be lost in transit or received corrupted.

Requirements:

  • Error Detection — Sender/receiver must detect error
  • Positive ACK — When receiver gets correct frame, it acknowledges
  • Negative ACK (NACK) — When receiver gets damaged frame, sends NACK
  • Retransmission — Sender sets timeout; if no ACK arrives before timeout, retransmits

ARQ Techniques:

  • Stop-and-Wait ARQ — Send one frame, wait for ACK; if timeout expires or NACK received, retransmit
  • Go-Back-N ARQ — Both sender and receiver maintain a window; sender can send N frames without waiting; if error in frame i, retransmit from frame i

Stop and Wait Protocol (PYQ 7 Marks)

Sender Receiver │ │ │──── Frame 0 ────────────────────▶│ │◀─── ACK 0 ──────────────────────│ │ │ │──── Frame 1 ────────────────────▶│ │◀─── ACK 1 ──────────────────────│ │ │ [Timer] ◀── Flow Control ──────────│ │ (After Consumption) │
  • Sender sends one frame at a time and starts a timer
  • Waits for acknowledgement before sending next frame
  • Each frame carries a CRC for error detection
  • Receiver sends ACK after receiving uncorrupted frame
  • A corrupted frame is NOT acknowledged

  • Offers both flow control and error control

MAC Layer — ALOHA, CSMA, CSMA/CD, CSMA/CA

IMP 7MPYQ ▾
MAC (Media Access Control) A sublayer of the data link layer. Responsible for flow control and multiplexing for transmission medium. Controls transmission of data packets via remotely shared channels. Sends data over the network interface card (NIC).

MAC Layer Position in OSI

Data Link Layer is divided into: ┌─────────────────────────────┐ │ LLC (Logical Link Control) │ ← handles flow control, error control, framing ├─────────────────────────────┤ │ MAC (Media Access Control) │ ← defines specific access method for each LAN └─────────────────────────────┘

Functions of MAC Layer

  • Provides abstraction of physical layer to LLC and upper layers
  • Encapsulates frames for transmission via physical medium
  • Resolves addressing of source and destination stations
  • Performs multiple access resolution (determines channel access method)
  • Performs collision resolution; initiates retransmission
  • Generates frame check sequences (protection against errors)

MAC Address

A unique identifier allotted to a NIC. Used as network address within a segment (Ethernet, Wi-Fi, Bluetooth). Hardwired/hardcoded in NIC. Format: 6 groups of 2 hexadecimal digits separated by colons.

Example: 00:0A:89:5B:F0:11

Multiple Access Protocols

┌──────────────────────────────────────┐ │ MULTIPLE ACCESS PROTOCOLS │ └────────────────┬─────────────────────┘ ┌────────┴────────┐─────────────────────┐ ▼ ▼ ▼ RANDOM ACCESS CONTROLLED ACCESS CHANNELIZATION • ALOHA • Reservation • FDMA • CSMA • Polling • TDMA • CSMA/CD • Token Passing • CDMA • CSMA/CA

A. ALOHA (PYQ 7 Marks)

ALOHA Designed for wireless LAN. Any station can transmit data simultaneously when a data frameset is available. Does not require carrier sensing.

ALOHA Rules:

  1. Any station can transmit to a channel at any time
  2. Does NOT require carrier sensing
  3. Collision and data frames may be lost
  4. Acknowledgement of frames exists (no collision detection)
  5. Requires retransmission after random backoff time

PURE ALOHA When data available, station transmits without checking if channel is idle. Waits for ACK. If no ACK within specified time, waits backoff time (Tb) then retransmits.
SLOTTED ALOHA Channel divided into fixed time slots. Station can ONLY send at beginning of a slot. Only one frame per slot allowed. Doubles efficiency of Pure ALOHA.

B. CSMA (Carrier Sense Multiple Access)

CSMA A protocol that SENSES the traffic on a channel (idle or busy) before transmitting. If channel is idle → transmit. If busy → wait until idle. Reduces chances of collision.

CSMA Access Modes:

  • 1-Persistent — If channel idle → immediately send (probability 1). If busy → wait and continuously sense; transmit as soon as idle.
  • Non-Persistent — If idle → immediately send. If busy → wait RANDOM time (not continuously), then sense again.
  • P-Persistent — Combination of 1-persistent and non-persistent. If idle → send with probability P; wait with probability (1-P). If busy → wait for next slot.
  • O-Persistent — Each station assigned a superiority order. If idle, each station waits for its turn to retransmit.

CSMA/CA (Collision Avoidance) Sends data; receives ACK to check channel is clear. Uses: Interframe Space (IFS) — wait time after channel idle; Contention Window — random slot wait; Acknowledgement — if ACK received, transmission successful.

Started in 1985 by IEEE Computer Society. A family of standards for LAN, PAN, and MAN. Maps to lower two layers (Physical + Data Link) of OSI. Divides Data Link layer into LLC and MAC sublayers.

Network Layer — Services & IP Addressing

IMPPYQ ▾

Network Layer Services (PYQ 7 Marks)

  • Packetizing — Encapsulating data from upper layers in network layer packet at source; decapsulating at destination. Source adds header with source/destination address. Routers cannot change source/destination address.
  • Routing — Process of moving data from one device to another. Network specifies strategies to find best route. Multiple routes available from source to destination.
  • Forwarding — Action applied by each router when a packet arrives at one of its interfaces. Router uses forwarding table to decide where to send packet.

  • Version (4 bits) — Set to 6 for IPv6
  • Traffic Class (8 bits) — Class or priority of packet; similar to TOS in IPv4; used in QoS
  • Flow Label (20 bits) — New in IPv6; differentiates traffic flows; flow label + source IP = unique data flow
  • Payload Length (16 bits) — Length of IPv6 payload in bytes (max 65535)
  • Next Header (8 bits) — Identifies type of first extension header or upper-layer protocol
  • Hop Limit (8 bits) — Same as TTL in IPv4; decremented by 1 at each hop; discarded at 0
  • Source Address (128 bits) — Address of packet originator
  • Destination Address (128 bits) — Address of packet recipient

Routing Algorithms — Distance Vector & Link State

IMP 7MPYQ ▾
DISTANCE VECTOR ROUTING A dynamic routing algorithm. Each router computes distance between itself and each possible destination (immediate neighbors). Shares knowledge of whole network to its neighbors. Sharing at regular intervals. Based on local knowledge.

Problems with Distance Vector:

  • Count to infinity problem (solved by splitting horizon)
  • Good news spreads fast, bad news spreads slowly
  • Persistent looping problem

Algorithm Steps:

  1. A router transmits its distance vector to each neighbor in a routing packet
  2. Each router receives and saves most recently received distance vector from each neighbor
  3. A router recalculates its distance vector when it receives a distance vector from a neighbor containing different information, or discovers a link has gone down
  4. DV calculation is based on minimizing the cost to each destination
LINK STATE ROUTING A dynamic routing algorithm. Each router shares knowledge of its NEIGHBORS (not whole network) with EVERY other router through flooding. Sharing only when there is a CHANGE. Based on global knowledge. Uses Dijkstra's algorithm.

Problems with Link State:

  • Heavy traffic due to flooding of packets
  • Flooding can result in infinite looping (solved by TTL field)

Link State Process:

  1. Each router learns about its own links (directly connected networks)
  2. Each router builds a Link-State Packet (LSP) containing state of each directly connected link
  3. Each router floods LSP to all neighbors; they store all LSPs in a database (LSDB)
  4. Each router uses database to construct complete map of topology
  5. Computes best path to each destination using Dijkstra's algorithm

Transport Layer — TCP vs UDP

IMP 7MPYQ ▾

Transport Layer Responsibilities (PYQ 7 Marks)

  • Process-to-Process Delivery — Uses Port Numbers (16-bit address) to identify client-server programs uniquely
  • End-to-End Connection — Creates connection between hosts using TCP or UDP
  • Multiplexing & Demultiplexing — Mux: multiple processes → one packet; Demux: one packet → multiple processes
  • Congestion Control — Open-loop (prevent) and closed-loop (remove); AIMD; Leaky bucket technique
  • Data Integrity & Error Correction — Error detection codes, checksums, ACK/NACK
  • Flow Control — Sliding window protocol; receiver sends window size to sender

Leaky Bucket Technique

Input rate can vary but output rate remains constant. Bursty chunks stored in bucket and sent out at average rate. Like a bucket with hole — water exits at fixed rate regardless of input rate.

TCP — Transmission Control Protocol

TCP Connection-oriented protocol. Establishes logical connection (handshake) before transferring data. Provides reliable delivery, flow control, error control, and congestion control.

Features of TCP:

  • Segment Numbering — Assigns byte numbers to data bytes, sequence numbers to segments
  • Connection-Oriented — Sender and receiver connected throughout; data order maintained
  • Full Duplex — Data can be sent in both directions simultaneously
  • Flow Control — Sliding window; limits rate of data transfer
  • Error Control — Byte-oriented; handles corrupted, lost, out-of-order, duplicate segments
  • Congestion Control — Takes into account congestion level in network

TCP Segment Format (PYQ 7 Marks)

┌─────────────────────────────────────────────────────────────┐ │ SOURCE PORT (16 bits) │ DESTINATION PORT (16 bits) │ ├─────────────────────────────────────────────────────────────┤ │ SEQUENCE NUMBER (32 bits) │ ├─────────────────────────────────────────────────────────────┤ │ ACKNOWLEDGEMENT NUMBER (32 bits) │ ├──────────┬────────────────────┬─────────────────────────────┤ │HLEN(4b) │ RESERVED (6b) │URG│ACK│PSH│RST│SYN│FIN│ ├──────────┴────────────────────┼─────────────────────────────┤ │ WINDOW SIZE (16 bits) │ CHECKSUM (16 bits) │ ├──────────────────────────────────────────────────────────────┤ │ URGENT POINTER (16 bits) │ OPTIONS + PADDING │ └──────────────────────────────────────────────────────────────┘ Header: 20 to 60 bytes
  • Source Port — Port number of application program in sending host
  • Destination Port — Port number of application program in receiving host
  • Sequence Number — Byte number of first byte of data in segment
  • Acknowledgement Number — Sequence number of next expected segment
  • HLEN — Header length in 4-byte units
  • Control Flags — 6 bits: URG, ACK, PSH, RST, SYN, FIN
  • Window Size — Defines window size of sending TCP (flow control)
  • Checksum — 16-bit checksum for error detection
  • Urgent Pointer — Valid only if URG flag is set

UDP — User Datagram Protocol

UDP Connectionless, unreliable protocol. No connection establishment prior to data transfer. Low-latency, loss-tolerating. No error checking. Faster than TCP.

UDP Header Format

┌──────────────────────────────────────────────┐ │ SOURCE PORT (16 bits) │ DEST. PORT (16 bits)│ ├──────────────────────────────────────────────┤ │ LENGTH (16 bits) │ CHECKSUM (16 bits) │ └──────────────────────────────────────────────┘ Header: Fixed 8 bytes

HTTP — Hyper Text Transfer Protocol & Persistent/Non-Persistent

IMP 7MPYQ ▾
HTTP Used to access data on the World Wide Web (www). Application layer protocol. Connectionless, media-independent, TCP/IP based, stateless protocol. Uses port 80. Transfers data as plain text, hypertext, audio, video, HTML files.
  • HTTP is simpler than FTP — uses only ONE connection (data connection; no control connection)
  • Similar to SMTP in terms of data transferred, but HTTP delivers messages immediately (SMTP stores and forwards)
  • Versions: HTTP 1.0, HTTP 1.1, HTTP 2.0, HTTP 3.0

HTTP Persistent vs Non-Persistent (PYQ 7 Marks)

PropertyNon-Persistent (HTTP 1.0)Persistent (HTTP 1.1)
TCP connectionOne TCP connection per request/responseSingle TCP connection for multiple requests/responses
Round Trip Time2 RTT for every objectOnly 1 RTT for all objects
CPU usageMore (multiple TCPs)Less (single TCP)
ModeNon-defaultDefault mode
Request methodsHEAD, POSTGET, HEAD, POST, PUT, DELETE
ObjectsMultiple connections neededSingle connection for all objects

Non-Persistent HTTP Steps:

  1. Client opens TCP connection and sends request
  2. Server sends response and closes connection
  3. Client reads data till end of file; closes connection

Persistent HTTP Steps:

  1. Client opens TCP connection and sends request
  2. Server sends response and leaves connection open
  3. Client can send any number of requests and get responses
  4. Server closes connection at client request or timeout

Persistent Pipelined vs Non-Pipelined:

  • Non-Pipelined — Client sends next request only after previous request acknowledged
  • Pipelined — Client can send new request even if previous request not yet acknowledged

Advantages of Persistent HTTP:

  • Saves CPU resources and time (connection opened only once)
  • Fast start for sending objects; less network congestion

Disadvantages of Persistent HTTP:

  • Wastes resources when no data being transmitted (connection stays open)
  • Less secure than non-persistent (connection always open)

FTP — File Transfer Protocol

IMP 7MPYQ ▾
FTP (File Transfer Protocol) Application layer protocol. Used to exchange files over the internet. Transfers both text and binary files. Uses TCP at transport layer. Connection-oriented. Out-of-band protocol (data and control info flow over different connections).

Why FTP? Systems may have different: file conventions, ways of representing text/data, directory structures. FTP overcomes these by establishing TWO connections between hosts.

FTP Basic Model (PYQ 7 Marks)

┌─────────────────────────────────────────────────────┐ │ USER │ │ │ │ │ ▼ │ │ USER INTERFACE │ │ │ │ │ ▼ Control Connection (Port 21) │ │ CONTROL PROCESS ──────────────────────────▶ CONTROL │ │ │ PROCESS │ │ ▼ Data Connection (Port 20) │ │ DATA TRANSFER ◀──────────────────────────▶ DATA │ │ PROCESS TCP/IP TRANSFER │ │ PROCESS │ │ CLIENT SERVER │ └─────────────────────────────────────────────────────┘

Two Types of Connections in FTP

  • Control Connection (Port 21) — Simple rules; transfers one line of command/response at a time; remains connected during entire FTP session
  • Data Connection (Port 20) — Complex rules (data types vary); opens when command for file transfer comes; closes when file is transferred

FTP Client Components: User Interface, Control Process, Data Transfer Process

FTP Server Components: Server Control Process, Server Data Transfer Process

Merits of FTP:

  • Speed — fastest way for file transfer
  • Efficient — does not need to complete all operations to get entire file
  • Security — protected with user credentials
  • Two-way transfer — back and forth movement of files

Demerits of FTP:

  • File size limit — 2 GB limit
  • Security — data is not encrypted
  • Multiple receiving not supported

Email — SMTP & Electronic Mail Architecture

IMPPYQ ▾
SMTP — Simple Mail Transfer Protocol Mainly used for sending emails efficiently over the internet. Push protocol. Uses TCP at transport layer. Uses persistent TCP connections (can send multiple emails at once). Connection-oriented, stateless, in-band protocol. Port 25.

SMTP Characteristics:

  • Pure text-based protocol — handles only 7-bit ASCII text
  • Cannot transfer images, video, audio directly
  • Cannot transfer executable files and binary objects
  • MIME (Multipurpose Internet Mail Extension) extends SMTP to handle graphics, audio, video
  • SMTP Auth — SMTP Authentication for authentication purposes

Email Message Components

  • Message Envelope — Email's electronic format
  • Message Header — Email subject line and sender/recipient information
  • Message Body — Images, text, and other file attachments

Electronic Mail Architecture (PYQ 3 Marks)

User A User B │ │ [UA] ←User Agent→ [UA] │ (Outlook, etc.) │ │ │ [MTA] ──SMTP──▶ [Relay MTA] ──SMTP──▶ [MTA] [Server] Client │ [MDA] │ ◀─POP3/IMAP─ User B retrieves
  • User Agent (UA) — Software package (e.g., Outlook). Creates, reads, replies to, forwards messages.
  • Message Transfer Agent (MTA) — Actual mail transfer. MTA client needed to send; MTA server needed to receive. SMTP is a MTA.
  • Message Access Agent (MAA) — To receive mail, MAA needed. Receiver pulls message from mail server. MAA client runs on receiver; MAA server on mail server.

Working of SMTP:

  1. Composition — User prepares mail through MUA
  2. Submission — MUA submits email to SMTP server using SMTP on port 25
  3. Delivery — MTA transfers email to appropriate server
  4. Storage — Server stores received mail in Mail Delivery Agent (MDA)
  5. Retrieval — Stored email in MDA retrieved by MUA (using POP3/IMAP)

DNS, WWW & Web Documents

IMPPYQ ▾
DNS — Domain Name System A directory service that provides mapping between the name of a host on the network and its numerical (IP) address. Allows users to use friendly names instead of remembering IP addresses. Application layer protocol. Uses UDP (Port 53).

DNS Resolution Process

  1. User enters domain name in browser
  2. Browser sends recursive DNS query to find IP address
  3. Query goes to recursive DNS server (usually managed by ISP); if it has address → returns it
  4. If not, queries: DNS root name servers → TLD name servers → authoritative name servers
  5. These servers redirect until DNS record with queried IP is retrieved
  6. Recursive server caches the A record for domain name (future queries answered directly)
  7. If authoritative server can't find info → returns error message

DNS Namespace

Set of all domain names registered in DNS. Organized into tree-like structure. Root domain at top. Below: Top-Level Domains (.com, .net, .org). Below TLDs: Second-level domains. Can have subdomains (e.g., www.example.com, mail.example.com). Managed by ICANN.

World Wide Web (WWW)

WWW (World Wide Web) A collection of information linked together from points all over the world. A way of exchanging information between computers on the Internet. Network of pages consisting of images, text, and sounds. Invented by Tim Berners-Lee at CERN.

Architecture of WWW: Distributed client/server service. Each website holds web pages. Each web page contains links to other pages.

Components of WWW:

  1. Structural: Clients/Browsers, Servers, Caches, Internet
  2. Semantic: HTTP, HTML, XML, URL

Three Types of Web Documents (PYQ 3 Marks)

STATIC WEB DOCUMENT Resides in a file associated with a web server. Contents determined at time of writing. Each request results in exactly the same response. Created using HTML, XML, XHTML.
DYNAMIC WEB DOCUMENT Does NOT exist in predefined form. When request arrives, web server runs application program that creates the document. Contents can vary from one request to another. Created using ASP, JSP, PHP scripting languages.
ACTIVE WEB DOCUMENT Consists of a computer program that the server sends to the browser, which the browser must run locally. Can interact with user and change display continuously. Example: Java applets, JavaScript.

URL — Uniform Resource Locator

Protocol :// Host : Port / Path http ://www.example.com:80/index.html
  • Protocol — Client/server program used to retrieve document (e.g., HTTP)
  • Host Computer — Computer on which information is located
  • Port — Optional; inserted between host and path, separated by colon
  • Path — Pathname of file where information is located

SSH — Secure Shell Protocol

IMP 7MPYQ ▾
SSH — Secure Shell An application layer program used for remote logging and secure file transfer. Creates secure encrypted connection between client and server. Runs on TCP Port 22.

SSH Components (PYQ 3 Marks)

┌─────────────────────────────────────────────────┐ │ SSH APPLICATIONS │ Application Layer ├──────────────────────────┬──────────────────────┤ │ SSH-CONN │ SSH-AUTH │ ├─────────────────────────────────────────────────┤ │ SSH-TRANS │ ├─────────────────────────────────────────────────┤ │ TCP (Transport Layer, Port 22) │ └─────────────────────────────────────────────────┘
  1. SSH-TRANS (SSH Transport Layer Protocol)
    • Establishes secure connection between SSH client and server over TCP

    • Authenticates SSH client for the server

  3. SSH-CONN (SSH Connection Protocol)
    • Allows multiple channels to run over the secure connection
    • Each channel can be used for different purpose
  4. SSH Applications
    • Remote console login, SFTP (Secure File Transfer Protocol)

Peer-to-Peer Architecture vs Client-Server

PYQ ▾
PEER-TO-PEER (P2P) ARCHITECTURE Each workstation has the same capabilities and responsibilities. Each station can act as BOTH client and server. Responsibility is shared among peers. All peers contribute and consume resources. No centralized server needed.

Characteristics of P2P:

  • Often created by collections of 12 or fewer machines
  • Nodes both consume and produce resources — as nodes grow, capability grows (unlike client-server where server gets overloaded)
  • Difficult to provide security (nodes act as both client and server — risk of denial of service attack)
  • Easily scalable and cost effective
  • Each computer manages itself — easy to set up and maintain
