📡 Data Communication — Definition & Components
1M3M
▾
Definition Data communication is the exchange of data between two devices via some form of transmission medium (such as a wire or cable). The communicating devices must be part of a communication system made up of hardware and software.
5 Components of a Data Communication System
  • Message — The information to be communicated (text, numbers, images, audio, video)
  • Sender — The device that sends the data (computer, phone, camera)
  • Receiver — The device that receives the message (computer, TV, phone)
  • Transmission Medium — The physical path (twisted-pair wire, coaxial cable, fiber-optic, radio waves)
  • Protocol — A set of rules that govern data communications. Without a protocol, two devices may be connected but not communicating.
4 Characteristics of Effective Data Communication
  • Delivery — Data must reach the correct destination only
  • Accuracy — Data must be delivered without alteration
  • Timeliness — Data must be delivered on time (real-time for audio/video)
  • Jitter — Variation in packet arrival time; uneven delay in audio/video packets

4 Network Topologies (7 Mark Answer)
  • Mesh — Every device has dedicated point-to-point link to every other device. Links needed = n(n-1)/2. Advantages: Robust, private, easy fault isolation. Disadvantages: Very expensive, difficult installation.
  • Star — Each device connects only to a central hub. No direct device-to-device links. Advantages: Less expensive, easy install, if one link fails only that link affected. Disadvantages: If hub fails, entire network fails.
  • Bus — One long backbone cable connects all devices via drop lines and taps. Multipoint topology. Advantages: Easy installation, less cabling. Disadvantages: Difficult fault isolation, hard to add new devices.
  • Ring — Each device connects to only two adjacent devices. Signal passes in one direction. Each device has a repeater. Advantages: Easy to install/reconfigure. Disadvantages: Break in ring disables entire network.

⚡ Transmission Impairment — Attenuation, Distortion, Noise
HOT3M
▾
Definition Transmission impairment occurs when the received signal is not identical to the transmitted signal. The quality of the signal gets destroyed, making it difficult to interpret at the receiver.
  • Attenuation — Loss of energy. The strength of the signal decreases as it travels a longer distance through the medium. Signal becomes weaker.
  • Distortion — Change in the shape or form of the signal. Occurs in composite signals (made of different frequencies). Each frequency has its own propagation speed, so they arrive at different times → delay distortion.
  • Noise — Unwanted signals added to the data during transmission. Types: Thermal noise (random electron motion), Induced noise (motors, appliances), Crosstalk (effect of one wire on another), Impulse noise (lightning, power lines).

🔌 Transmission Media — Guided & Unguided
HOT7M
▾
Definition Transmission media is a communication channel that carries information from sender to receiver via electromagnetic signals. It is the physical path between transmitter and receiver. Available at Physical Layer (Layer 1) of OSI.
Guided (Wired) Media
  • Twisted-Pair Cable — Two copper conductors twisted together. One wire carries signal; other is ground. Most common connector: RJ-45. Used in: telephone lines, LANs. Twisting reduces interference. Types: UTP (Unshielded) and STP (Shielded).
  • Coaxial Cable — Central copper core conductor enclosed in insulating sheath, then outer metallic foil shield (against noise), then plastic cover. Most common connector: BNC. Used in: cable TV, Ethernet LANs.
  • Fiber-Optic Cable — Made of glass or plastic. Transmits signals as light pulses. Uses principle of total internal reflection. Glass core surrounded by cladding of less dense material.

🔗 Data Link Layer — Framing, Flow Control & Error Control
HOT7M
▾
Data Link Layer Services
  • Framing — Breaks the bit stream into manageable data units called frames
  • Physical Addressing — Adds MAC address to frame header
  • Flow Control — Prevents fast sender from overwhelming slow receiver
  • Error Control — Detects and corrects errors in frames
  • Access Control — Determines which device has control of shared medium at any time
Framing — Parts of a Frame
  • Frame Header — Contains source and destination MAC addresses
  • Payload field — Contains the actual message to be delivered
  • Trailer (CRC) — Contains error detection/correction bits
  • Flag — Marks the beginning and end of the frame
Types of Framing
  • Fixed-sized Framing — Frame size is fixed; length acts as delimiter. No extra bits needed to identify start/end.
  • Variable-sized Framing — Frame size differs. Needs mechanisms to mark frame boundaries:
    → Byte-Stuffing (Character-oriented): A special byte is stuffed in message to differentiate from delimiter
    → Bit-Stuffing (Bit-oriented): A pattern of bits stuffed in message to differentiate from delimiter
Flow Control
Definition Flow control is the process of managing the rate of data transmission between two nodes to prevent a fast sender from overwhelming a slow receiver.
  • Stop and Wait — Sender transmits ONE frame, then STOPS and waits for ACK before sending the next. Simple but inefficient (sender is idle while waiting).
  • Sliding Window — Both sender and receiver agree on a number of frames (window size). Sender can send multiple frames before waiting for ACK. More efficient, better resource utilization.
Stop and Wait Protocol (7 Mark Answer)
Sender Receiver | | |──── Frame 0 ──────────────▶| (Sender starts timer) |◀─── ACK 0 ─────────────────| (Receiver sends ACK) | | |──── Frame 1 ──────────────▶| |◀─── ACK 1 ─────────────────| | | [TIMER EXPIRES → Retransmit] ← (If no ACK received)
  • Sender sends one frame at a time and starts a timer
  • Each frame carries a CRC for error detection
  • Receiver sends ACK after receiving an uncorrupted frame
  • A corrupted frame is NOT acknowledged
  • If ACK received before timer → send next frame
  • If timer expires → retransmit the same frame (assumes frame was lost)
  • Provides both flow control AND error control
Error Control Requirements
  • Error Detection — Sender/receiver must detect the error
  • Positive ACK — Receiver sends ACK when it receives a correct frame
  • Negative ACK (NACK) — Receiver sends NACK when it receives a damaged/duplicate frame
  • Retransmission — If no ACK arrives before timeout, sender retransmits the frame

📶 MAC Layer — ALOHA, CSMA, CSMA/CD, CSMA/CA
HOT7M
▾
MAC — Media Access Control A sublayer of the Data Link Layer. Responsible for flow control and multiplexing for the transmission medium. Controls how devices share and access the communication channel. It sends data over the network interface card (NIC).
ALOHA Protocol (7 Mark Answer)
ALOHA A random access protocol designed for wireless LAN. Any station can transmit data to a channel at any time without checking if the channel is busy. Does NOT require carrier sensing. Collisions may occur.
PropertyPure ALOHASlotted ALOHA
Transmission timeAny timeOnly at beginning of a time slot
Vulnerable time2 × TfrTfr (one slot only)
Max Throughput18.4% (when G = 1/2)37% (when G = 1)
EfficiencyLowerDoubles Pure ALOHA
  • Pure ALOHA Rules: Any station transmits whenever data is ready. Waits for ACK. If no ACK → waits a random backoff time (Tb) → retransmits.
  • Slotted ALOHA: Channel divided into fixed time slots. Station can ONLY send at the beginning of a slot. One frame per slot allowed. Reduces collision probability.
CSMA — Carrier Sense Multiple Access
CSMA A protocol that SENSES the channel before transmitting. If channel is idle → transmit. If channel is busy → wait. Reduces chances of collision compared to ALOHA.
  • 1-Persistent — If idle → immediately send (probability = 1). If busy → continuously sense until idle, then send immediately.
  • Non-Persistent — If idle → send. If busy → wait a RANDOM time, then sense again.
  • P-Persistent — If idle → send with probability P, wait with probability (1-P). Combination of the above two.

🌍 Network Layer — Services & IP Addressing
HOT7M3M
▾
Three Main Network Layer Services
  • Packetizing — Encapsulating data received from upper layers into a network-layer packet at the source. Decapsulating at the destination. The source adds a header with source/destination address. Routers CANNOT change source or destination address.
  • Routing — Process of moving data from one device to another. Network specifies strategies to find the best possible route. Multiple routes may exist from source to destination. Uses routing protocols like RIP.
  • Forwarding — Action applied by each router when a packet arrives at one of its interfaces. Router checks the forwarding/routing table and decides where to send the packet next.

IPv6 Header Fields Explained
  • Version (4b) — Set to 6 for IPv6
  • Traffic Class (8b) — Class/priority of packet. Similar to TOS in IPv4. Used in QoS control.
  • Flow Label (20b) — NEW in IPv6. Differentiates traffic flows. Flow label + source IP = unique data flow.
  • Payload Length (16b) — Length of IPv6 payload in bytes (max 65535)
  • Next Header (8b) — Identifies the type of the first extension header or upper-layer protocol
  • Hop Limit (8b) — Similar to TTL in IPv4. Decremented by 1 at each hop. Packet discarded when it reaches 0.
  • Source Address (128b) — Address of the packet originator
  • Destination Address (128b) — Address of the packet recipient

🗺️ Routing Algorithms — Distance Vector & Link State
HOT7M
▾
Distance Vector Routing
Distance Vector Routing A dynamic routing algorithm where each router computes the distance between itself and each possible destination (immediate neighbors). The router shares knowledge about the WHOLE NETWORK with its NEIGHBORS. Sharing happens at REGULAR INTERVALS. Based on LOCAL knowledge.
  • Each router maintains a table of best-known distances to each destination
  • Tables updated based on information from neighbors
  • Uses Bellman-Ford algorithm
  • Problems: Count-to-infinity problem (solved by split horizon); Good news spreads fast, bad news spreads slowly; Persistent looping
Link State Routing
Link State Routing A dynamic routing algorithm where each router shares knowledge of its NEIGHBORS with EVERY OTHER ROUTER in the network through flooding. Information sharing only when there is a CHANGE. Based on GLOBAL knowledge. Uses Dijkstra's algorithm.
  • Each router builds a Link-State Packet (LSP) about its directly connected links
  • Floods LSP to all other routers; all routers store LSPs in a database (LSDB)
  • Each router constructs a complete map and computes best path using Dijkstra's algorithm
  • Problems: Heavy traffic due to flooding; Flooding can cause infinite looping (solved by TTL field)

🚚 Transport Layer — TCP, UDP & Segment Format
HOT7M3M
▾
Transport Layer Responsibilities
  • Process-to-Process Delivery — Uses Port Numbers (16-bit) to identify which application program the data belongs to
  • End-to-End Connection — Creates connection between hosts using TCP or UDP
  • Multiplexing & Demultiplexing — Mux: combines data from multiple processes into one packet. Demux: distributes received packet to correct process.
  • Congestion Control — Prevents too much data from entering network. Uses AIMD and Leaky Bucket technique.
  • Data Integrity & Error Correction — Uses checksums, ACK/NACK services
  • Flow Control — Uses sliding window protocol
TCP Segment Format (7 Mark Answer — Draw Diagram)
┌───────────────────────────────────────────────────────────┐ │ SOURCE PORT (16 bits) │ DESTINATION PORT (16 bits) │ ├───────────────────────────────────────────────────────────┤ │ SEQUENCE NUMBER (32 bits) │ ├───────────────────────────────────────────────────────────┤ │ ACKNOWLEDGEMENT NUMBER (32 bits) │ ├──────────┬────────────────────┬──────────────────────────┤ │ HLEN(4b) │ RESERVED (6b) │ URG|ACK|PSH|RST|SYN|FIN │ ├──────────┴────────────────────┼──────────────────────────┤ │ WINDOW SIZE (16 bits) │ CHECKSUM (16 bits) │ ├───────────────────────────────┴──────────────────────────┤ │ URGENT POINTER (16 bits) │ OPTIONS + PADDING │ └───────────────────────────────────────────────────────────┘ Header size: 20 to 60 bytes

🌐 HTTP — Persistent & Non-Persistent Connections
HOT7M
▾
HTTP — HyperText Transfer Protocol Used to access data on the World Wide Web. It is an Application Layer protocol. It is connectionless, media-independent, TCP/IP based, and stateless. Uses Port 80. Transfers data as plain text, hypertext, audio, video, HTML files. Versions: HTTP 1.0, 1.1, 2.0, 3.0.
Key HTTP Facts
  • HTTP uses only ONE connection (data connection only; unlike FTP which uses two)
  • HTTP delivers messages immediately (unlike SMTP which stores and forwards)
  • HTTP is simpler than FTP because there is no separate control connection

📂 FTP — File Transfer Protocol
HOT7M
▾
FTP — File Transfer Protocol An application layer protocol used to exchange files over the internet. Transfers both text and binary files. Uses TCP at transport layer. Connection-oriented. Out-of-band protocol — data and control information flow over DIFFERENT connections.
Why FTP is needed?

Different systems may have: different file naming conventions, different ways of representing text/data, different directory structures. FTP overcomes these by establishing TWO separate connections.

Two Types of Connections in FTP
  • Control Connection (Port 21) — Uses simple rules. Transfers one command/response at a time. Stays connected during the ENTIRE FTP session. Made between control processes.
  • Data Connection (Port 20) — Uses complex rules (data types vary). Opens when a file transfer command comes. Closes when the file transfer is complete.

📧 Email — SMTP & Email Architecture
HOT7M3M
▾
SMTP — Simple Mail Transfer Protocol Mainly used for sending emails efficiently over the Internet. It is a push protocol. Uses TCP at transport layer. Uses persistent TCP connections (can send multiple emails at once). Connection-oriented, stateless, in-band protocol. Uses Port 25.
Key SMTP Facts
  • Pure text-based protocol — handles only 7-bit ASCII text
  • CANNOT directly transfer images, video, audio, or binary files
  • MIME (Multipurpose Internet Mail Extensions) extends SMTP to handle multimedia (graphics, audio, video files)
  • MIME was specially designed for SMTP
Email Message Components
  • Message Envelope — Email's electronic format (routing information)
  • Message Header — Subject line and sender/recipient information
  • Message Body — Images, text, and other file attachments
Email Architecture — Components (3 Mark Answer)
  • User Agent (UA) — Software package (e.g., Microsoft Outlook). Creates, reads, replies to, and forwards email messages. The user interacts through the UA.
  • Message Transfer Agent (MTA) — Handles actual mail transfer across the internet. To send: MTA client needed. To receive: MTA server needed. SMTP is an MTA protocol.
  • Message Access Agent (MAA) — Allows receiver to pull messages from the mail server. MAA client runs on receiver's machine; MAA server runs on mail server. Uses POP3 or IMAP protocols.

DNS — Domain Name System A directory service that provides a mapping between the name of a host (like www.google.com) and its numerical IP address. It allows users to use friendly names instead of remembering IP addresses. Uses UDP on Port 53.
DNS Resolution — Step by Step
  • User enters a domain name (e.g., www.example.com) in browser
  • Browser sends a recursive DNS query to find the IP address
  • Query goes to recursive DNS server (usually ISP's server). If it has the address → returns it directly.
  • If not found, queries in order: DNS Root Name Servers → TLD Name Servers (.com, .org) → Authoritative Name Servers
  • Authoritative server returns the IP address to the recursive server
  • Recursive server caches the A record. Returns IP to browser. Webpage loads.
  • If authoritative server cannot find info → returns error message

🔐 SSH — Secure Shell Protocol
HOT7M3M
▾
SSH — Secure Shell An application layer program used for remote logging and secure file transfer. Creates a secure encrypted connection between a client and server over TCP. Runs on Port 22. It is a secure alternative to Telnet.
SSH 4 Components (3M or 7M Answer)
┌───────────────────────────────────────────┐ │ SSH APPLICATIONS │ ← Application Layer ├──────────────────────┬────────────────────┤ │ SSH-CONN │ SSH-AUTH │ ├───────────────────────────────────────────┤ │ SSH-TRANS │ ├───────────────────────────────────────────┤ │ TCP (Transport Layer, Port 22) │ └───────────────────────────────────────────┘
  • SSH-TRANS (Transport Layer Protocol) — Establishes secure connection between SSH client and server over TCP. Negotiates security parameters (encryption algorithm, HMAC). Creates secure tunnel. Provides: privacy, data integrity, server authentication, compression.
  • SSH-AUTH (Authentication Protocol) — Authenticates the SSH client to the server. After secure channel is established, server needs to verify the client's identity.
  • SSH-CONN (Connection Protocol) — Allows multiple logical channels to run over the single secure connection. Each channel can be used for a different purpose.
  • SSH Applications — Once secure connection is established, SSH allows different application programs to use it. Examples: Remote console login, SFTP (Secure File Transfer Protocol).
